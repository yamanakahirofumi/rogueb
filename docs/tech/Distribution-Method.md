# 配布方法

本プロジェクトでは、エンドユーザーが別途 Java ランタイム（JRE）をインストールすることなく実行できるよう、カスタム JRE を含めた形式で配布します。

## 1. 配布形式
- **カスタム JRE の同梱**：`jlink` および `jpackage` を活用し、アプリの実行に必要なモジュールのみを抽出した最小限のランタイムを同梱します。
- **実行形式**：
    - **インストーラー**：OS ごとのインストーラー（例：.msi、.dmg、.deb/rpm）を提供します。
    - **ZIP 配布**：`jlink` で作成したカスタム JRE を含め、解凍するだけで実行可能な ZIP アーカイブ形式でも提供します。

## 2. メリット
- **導入コストの低減**：ユーザー側での環境構築（Java のインストールや環境変数の設定）が不要になります。
- **動作の安定性**：開発時に検証した特定の JRE バージョンが同梱されるため、実行環境による依存関係トラブルを回避できます。
- **軽量化**：フルセットの JDK/JRE ではなく、必要なモジュールのみを抽出するため、配布サイズを抑制できます。

## 3. 関連ツール
- [jlink](https://docs.oracle.com/en/java/javase/21/docs/specs/man/jlink.html)：カスタム・ランタイム・イメージの作成
- [jpackage](https://docs.oracle.com/en/java/javase/21/docs/specs/man/jpackage.html)：自己完結型アプリケーション・パッケージの作成

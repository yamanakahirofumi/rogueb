# ロギング方針

デバッグおよび保守のため、適切なロギングを行います。

## 1. 基本方針
本プロジェクトでは、Java 標準のロギング API を使用して、アプリケーションの動作状況や例外発生時の情報を記録します。

## 2. 使用ライブラリ
- **ライブラリ**：`java.util.logging` (標準)

## 3. ログレベルの使い分け
ログレベルは以下のように使い分けます。

- **INFO**：
    - アプリケーションの主要なイベント（ゲーム開始、リソース読み込み成功、ワールド生成完了など）。
    - 正常系の重要な進行状況を記録します。
- **WARNING**：
    - 続行は可能だが注意が必要な事象（サポート外の画像形式の読み込み試行、ユーザーによる不適切な操作など）。
- **SEVERE**：
    - 処理の続行が困難な予期せぬ実行時エラー、必須リソースの読み込み失敗。

## 4. 出力内容
- 発生日時
- ログレベル
- 発生箇所（クラス名、メソッド名）
- メッセージ（および例外発生時はスタックトレース）

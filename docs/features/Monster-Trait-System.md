# モンスター特性システム (Monster Trait System)

## 1. 概要
本ドキュメントは、モンスターが種族固有、あるいは個体ごとに保持するパッシブ能力「特性（Trait）」の仕様を定義します。特性は、戦闘中のステータス補正や、環境・状態異常に対する耐性、特殊な行動ロジックなどを提供します。

## 2. 特性の基本仕様
特性は、スキル（能動的なアクション）とは異なり、条件を満たしている間、あるいは常に自動で効果を発揮します。

- **保持場所**:
    - `MonsterDomain.traits`: その種族が標準で持っている特性。
    - `MonsterInstanceDomain.traits`: その個体が現在保持している特性。
- **重複設定**: 同一の特性を複数保持することはできません。

### 2.1 特性の累積と優先順位 (Trait Stacking & Priority)
複数の異なる特性、または特性とシステム（忠誠度等）が同じパラメータに影響を与える場合のルールは以下の通りです。

- **同系統の特性（上位互換）**: `REGENERATION` と `REGENERATION_II` のように、同じ効果で強度が異なる特性を保持している場合、**より強力な方（II）のみ**が適用されます。
- **乗算される効果**: ダメージ軽減（`SLIME_BODY`）や攻撃力上昇（`AGGRESSIVE`）などは、原則として**乗算**されます。
- **独立した判定**: `BRACE` 特性と忠誠度による「食いしばり」は、それぞれ個別に確率判定が行われます（一方が失敗しても他方が成功すれば生き残る）。
- **耐性の優先**: `FIRE_IMMUNITY`（無効）と `SLIME_BODY`（火属性 1.5 倍）が共存する場合、**無効化が最優先**されます。

## 3. 主要特性リスト
システムで標準的に定義される特性の ID と効果の一覧です。

| 特性 ID | 名称 | カテゴリ | 効果概要 | 備考 |
| :--- | :--- | :---: | :--- | :--- |
| `REGENERATION` | 自己再生 | 能力強化 | `subStep % 10 == 0` のタイミングで、最大 HP の 3% を回復する。 | 自然回復（1%）とは別に累積して発動。 |
| `MIASMA_RESISTANCE` | 瘴気耐性 | 耐性・無効 | 環境効果「瘴気」によるダメージおよび自然回復無効化を無視する。 | ランク A 以上のダンジョンで有効。 |
| `FIRE_IMMUNITY` | 火属性無効 | 耐性・無効 | 火属性によるダメージを無効化する。 | |
| `BRACE` | 食いしばり | 特殊・行動 | HP 2 以上の時に致死ダメージを受けても、10% の確率で HP 1 で耐える。 | 忠誠度による効果と重複（個別判定）可能。 |
| `SLIME_BODY` | スライム体質 | 特殊・行動 | 物理ダメージを 20% 軽減するが、火属性ダメージが 1.5 倍になる。 | スライム系の固有特性。 |
| `UNDEAD_SOUL` | 不死の魂 | 耐性・無効 | 状態異常「睡眠」「封印」「毒」を無効化する。聖なる（Holy）属性の攻撃から **1.5 倍** のダメージを受ける。 | アンデッド系の固有特性。 |
| `FLIGHT` | 飛行 | 特殊・行動 | 「落とし穴」「地雷」などの一部のトラップを無効化する。 | 地形による移動コストも無視。 |
| `AGGRESSIVE` | 闘争本能 | 能力強化 | 自身の HP が 30% 以下の時、物理攻撃力および魔法攻撃力が 1.2 倍になる。 | ダメージ計算時の Stat 値に乗算。 |
| `SCAVENGER` | スカベンジャー | 特殊・行動 | プレイヤーを撃破した際、追加で獲得ゴールドが 20% 上昇する。 | |
| `STATUS_IMMUNITY` | 全状態異常無効 | 耐性・無効 | すべての状態異常（毒、麻痺、睡眠等）にかからなくなる。 | ボスモンスター向けの強力な特性。 |
| `REGENERATION_II` | 自己再生 II | 能力強化 | `subStep % 10 == 0` のタイミングで、最大 HP の 5% を回復する。 | ランク S の賞金稼ぎ等が保持。 |
| `TRACKING` | 追跡 | 特殊・行動 | 視界内・外に関わらず、常にプレイヤーの現在地を把握し最短距離で追跡する。 | AI の索敵ロジックをバイパスする。 |
| `SEE_INVISIBILITY` | 不可視看破 | 特殊・行動 | 状態異常「不可視」状態のユニットを視認できる。 | 対象の「不可視」による隠蔽効果を無効化。 |

## 4. 特性の獲得と継承

### 4.1 種族特性
各種族（`MonsterDomain`）は、0 〜 3 つ程度の標準特性を持っています。その種族の個体が生成された際、これらは自動的に `MonsterInstanceDomain.traits` にコピーされます。

### 4.2 突然変異 (Mutation)
[モンスター繁殖システム](./Monster-Breeding-System.md) において、稀に両親が持っていない特性が発現する場合があります。
- **発現確率**: 5% 程度。
- **対象**: そのカテゴリ（Type）に関連する特性、または汎用的な特性から抽選されます。

### 4.3 進化による変化
[モンスター進化システム](./Monster-Evolution-System.md) によって種族が変化した際、特性も進化後の種族のものへと更新されます。ただし、進化前の個体が「突然変異」等で得た固有の特性は、そのまま引き継がれる場合があります。

## 5. 実装時の考慮事項
- **計算順序**:
    - ステータス補正型（`AGGRESSIVE` 等）：ダメージ計算式の `Stat` 値に対して直接乗算します。
    - ダメージ補正型（`SLIME_BODY` 等）：基本ダメージ算出後、防御力による減算の前に適用します。
    - 詳細は [戦闘システム](./Combat-System.md) を参照してください。
- **回復処理の同期**: `REGENERATION` 系の回復は、自然回復と同じ `subStep % 10 == 0` のタイミングで行うことで、処理負荷を分散し計算を簡略化します。
- **AI ロジック**: `TRACKING` を持つ個体は、通常の「視界内にプレイヤーがいれば追跡、いなければランダム移動」というアルゴリズムではなく、常に「プレイヤーの座標を目指して最短経路で移動」する専用のルーチンを実行します。
- **表示**: クライアント側のステータス画面において、保持している特性の名称と説明が確認できるようにします。

## 6. 今後の拡張
- **特性強化**: 特定のアイテムを使用して、既存の特性を強化（例：自己再生 I → II）する仕組み。
- **連携特性**: パーティ内に特定の組み合わせの特性を持つモンスターがいる場合に発動するシナジー効果。

# ダンジョン生成システム (Dungeon Generation System)

## 1. 概要
本ドキュメントは、ランダム生成ダンジョン（フロンティア）におけるフロア構造、アイテム、およびモンスターの自動生成アルゴリズムの仕様を定義します。生成プロセスは再現性を確保するため、[Dungeonモジュール ドメインモデル](./domain_models/Dungeon.md) で定義されたシード値に基づきます。

## 2. 生成アルゴリズムのバリエーション
ダンジョンのフロアごとに、以下のいずれかのアルゴリズムが選択されます。

### 2.1 部屋・通路型 (Standard Room & Corridor)
最も標準的なローグライクの構造です。
- **特徴**: 複数の矩形の部屋が通路で結ばれた構造。
- **生成プロセス**:
    1. フロアを複数の区画（グリッド）に分割する。
    2. 各区画内にランダムなサイズの部屋を作成する。
    3. 隣接する部屋同士を直線または L 字型の通路で連結する。
- **制御パラメータ**: `roomCountSeed` により、部屋の最小/最大数およびサイズが決定されます。

### 2.2 洞窟型 (Natural Cave)
自然な岩肌や空洞を模した、曲線的で不規則な形状を持つ構造です。

- **特徴**: 直線的な通路がなく、大小さまざまな広場とそれらを繋ぐ不規則な隙間（通路）で構成されます。見通しが悪い一方で、敵から身を隠せる凹凸が多く存在します。
- **生成プロセス（セル・オートマトン法）**:
    1. **初期配置**: フロアの各タイルに対して、指定された確率（標準: 45%）で壁（Wall）、残りを床（Floor）としてランダムに初期配置します。この割合は `roomCountSeed` をもとに若干の揺らぎ（±5%）を持たせます。
    2. **状態更新（反復処理）**: 近傍8マス（ムーア近傍）の壁タイルの数を $W$ とします。
        - **第1〜4世代（平滑化）**: 各タイルにおいて $W \ge 5$ の場合は壁、それ以外は床へと更新します。これにより、微細なノイズを排除し、大きな床と壁の塊を作ります。
        - **第5〜8世代（構造安定）**: 各タイルにおいて $W \ge 5$ または $W = 0$ の場合に壁とし、それ以外を床とします。これにより、孤立した1マスの床・壁の発生を防ぎ、より洞窟らしい締まりのある空間に仕上げます。
    3. **接続性の確保（Flood Fill & 連結トンネル）**:
        - 生成完了後、**塗りつぶし法（Flood Fill）**を用いて、全ての床タイルの連結コンポーネント（空洞）を特定します。
        - 最も面積が大きいメイン空洞を基準とし、他の小さな孤立空洞を探索します。
        - 孤立空洞が存在する場合、その中心座標からメイン空洞の最も近い床タイルに向けて、壁を貫通する「掘削通路（幅1マス）」をA*アルゴリズムを用いて作成し、全体の通過可能性を保証します。
        - 面積が極端に小さい孤立空洞（例: 4マス未満）は、ゲーム性向上のため床から壁へと埋め戻します。
- **人間が考えるべき設計（探索の楽しさ・プレイフィール）**:
    - 広場と狭い通路の「緩急」を意識したマップ構成とします。
    - 自然物を模した装飾として、空洞の隅や行き止まりに、水溜まり（Waterway）や結晶化した鉱床（アイテム配置の優先ポイント）を確率で配置し、探索したくなる誘因を作ります。

### 2.3 迷路型 (Maze)
複雑な分岐を持つ通路主体の網目状構造です。

- **特徴**: 部屋がほとんど存在せず、フロア全体が幅1マスの細い通路で覆われています。角が多く、1歩先が見通せないため、不意の遭遇戦が発生しやすい高緊張感マップです。
- **生成プロセス（棒倒し法 / 穴掘り法）**:
    1. **初期グリッド**: フロア全体を壁で埋め尽くし、奇数座標（XおよびYがともに奇数）を基準点（床）として配置します。
    2. **棒倒しによる通路形成**: 各基準点から、ランダムな方向（上、下、左、右）に壁を1マス伸ばします。最上段の基準点からは4方向、それ以降の基準点からはすでに壁が伸ばされている「上」を除く3方向に壁を伸ばし、迷路を形成します。
    3. **「ループ（環状路）」の自動生成（デッドエンド緩和）**:
        - 完全な迷路（ツリー構造）は行き止まりが多く、プレイヤーが引き返す（バックトラック）際に退屈さやストレスを感じやすいという問題があります。
        - これを解決するため、生成された迷路から「行き止まり（周囲3マスが壁である床タイル）」をすべて特定します。
        - 特定した行き止まりの壁（隣接する壁1マス）を、**確率（標準: 30%）で破壊**して他の通路と接続し、意図的に「環状の迂回経路（ループ）」を形成します。
- **人間が考えるべき設計（探索の楽しさ・プレイフィール）**:
    - ループを適度に設けることで、強力なモンスターに追跡された際に「迷路の角を回って迂回して逃げ切る」といった戦術的な立ち回りを可能にします。
    - 迷路の突き当たり（ループ化しなかった行き止まり）には、トラップ（隠蔽状態）または価値の高いアイテムを高い確率で配置し、「行き止まりまで歩いた労力」に対するリスクとリターンを設定します。

### 2.4 大部屋型 (Big Room)
フロア全体が仕切り壁のない一つの巨大な空間となっている特殊な構造です。

- **特徴**: 遮蔽物が極めて少なく、フロアのほぼ全域に視界が通ります。遠距離攻撃が飛び交い、一度に大量のモンスターに感知されて囲まれるリスク（モンスターハウス的なスリル）があります。
- **生成プロセス**:
    1. **フロア全開放**: 外周1マスのみを強固な壁（Strong Wall）で囲み、内部の全タイルを床にします。
    2. **障害物・ハザードの配置**:
        - **遮蔽柱のランダム配置**: フロア内部の総面積に対して 5% 〜 10% の割合で、1x1 または 2x2 の「柱（壁タイル）」をランダムに配置します。これにより、プレイヤーが敵の視界や直線スキルをカットするための射線遮蔽物として利用できるようにします。
        - **蛇行水路（Waterway）の配置**: フロアを大きく2つの領域に分断するように、幅1〜2マスの蛇行する水路を配置します。飛行（FLIGHT）特性を持たないプレイヤーや地上型モンスターの移動を制限し、水路を跨ぐ1マスの「橋（床）」を戦略的チョークポイントとします。
        - **溶岩だまり（Lava）の配置**: フロアの数箇所に 3x3 から 5x5 サイズの「溶岩タイル」を確率で配置します。進入すると10ダメージを受けるため、位置取りの制限や、スキル・魔法によるノックバックで敵を突き落とすギミックとして機能します。
- **人間が考えるべき設計（探索の楽しさ・プレイフィール）**:
    - 遮蔽物がないため、突入時の初期配置次第で一瞬でハメ殺される可能性があります。これを防ぐため、プレイヤーの初期スポン地点（上り階段）の周囲5×5マス以内には、モンスターやトラップ、溶岩を配置しないセーフティエリアを保証します。
    - 柱の陰に隠れながら弓や魔法で各個撃破する、または水路を背にして挟撃を防ぐなど、「地形を味方につける」プレイングの面白さを強調します。

### 2.5 難易度・出現規則 (Difficulty & Selection Curve)
ゲーム進行における飽き防止と難易度曲線の管理のため、フロアタイプはダンジョンの深さ（階層）および[ダンジョンランク](./Dungeon-Rank-System.md)に基づいて動的に決定されます。

- **Dungeon Rank C以下 / フロア 1〜3階 (初級)**:
    - 100% 「部屋・通路型」を適用します。ゲームシステムの基本（視界ルール、部屋と通路の概念）を学習するためのチュートリアル期間として機能させます。
- **Dungeon Rank B以上 / フロア 4〜9階 (中級)**:
    - 「部屋・通路型」をベース（70%）としつつ、「洞窟型」が30%の確率で出現するようになります。曲線的な地形での戦闘や位置取りの重要性が増します。
- **Dungeon Rank A以上 / フロア 10階以上 (上級・フロンティア)**:
    - 「部屋・通路型」（40%）、「洞窟型」（30%）、「迷路型」（20%）、「大部屋型」（10%）の割合でランダムに選定されます。
    - **大部屋型の出現制限**: 大部屋型は戦術的な難易度が極めて高いため、同一のダンジョン内で連続して大部屋型が選ばれることは避け、最大でも1つのダンジョンにつき1フロアのみに制限されます。

## 3. シード値による制御 (Seed-based Control)
同一のシード値からは常に同一のダンジョンが生成されるよう、以下の役割分担を行います。

| シード値 | 制御対象 |
| :--- | :--- |
| **`roomCountSeed`** | アルゴリズムの選択、部屋の数、部屋のサイズ、通路の接続パターン。 |
| **`itemSeed`** | アイテム（`thingList`）の種類、個数、および配置座標。 |
| **`monsterSeed`** | モンスター（`monsterList`）の種族、レベル、および初期配置座標。 |
| **`trapSeed`** | トラップ（`trapList`）の種類、および配置座標。 |

## 4. フロアサイズの規格 (Floor Dimension Standards)
ダンジョンの各フロアの面積（幅 × 高さ）は、[ダンジョンランクシステム](./Dungeon-Rank-System.md) における「最大配置コスト」の算出に直接影響します。ゲームバランスを維持するため、フロアサイズは以下の基準を考慮して決定される必要があります。

- **配置コストとの相関**: [ダンジョンランクシステム](./Dungeon-Rank-System.md) の算出式に基づき、フロアの面積が増加すると最大配置コストも上昇します。
- **密度のバランス**: フロアが広すぎる場合、配置されるモンスターの密度が希薄になり探索の緊張感が失われる可能性があります。逆に狭すぎる場合は、配置コストの上限により強力なモンスターを少数しか配置できなくなります。
- **生成アルゴリズムの適用**: 「部屋・通路型」や「洞窟型」など、選択されたアルゴリズムが想定する標準的な面積範囲（小規模な試練から大規模なフロンティアまで）に収まるように設計します。

## 5. 通過可能性の保証 (Connectivity Assurance)
すべてのダンジョン生成において、以下の「到達可能性」を保証する必要があります。

- **必須経路**: 上り階段（入り口）から下り階段（出口）まで、エンティティが通行可能なタイルのみを通って到達できること。
- **孤立の解消**: アイテムやモンスターが配置されるすべての部屋は、主要な経路から到達可能であること。
- **検証アルゴリズム**: 生成完了直後に、A* アルゴリズムまたは塗りつぶし法（Flood Fill）を用いて、上り階段から全重要ポイントへの経路を確認します。経路が遮断されている場合は、壁を破壊して通路を作成します。

## 6. オブジェクトの配置ロジック

### 6.1 階段の配置
- **上り階段**: 最初の部屋、または洞窟の端に配置されます。
- **下り階段**: 上り階段から一定以上のマンハッタン距離（例: フロア幅の 50% 以上）離れた位置に配置されます。

### 6.2 アイテム・金の配置
- `itemSeed` に基づき、部屋の内部または通路の行き止まりに優先的に配置されます。
- フロアあたりのアイテム総数は、フロアの面積 (`width * height`) に比例した上限値の範囲内で決定されます。

### 6.3 モンスターの配置
- `monsterSeed` に基づき、プレイヤーの開始地点（上り階段）から一定距離以上離れた場所に配置されます。
- 部屋の中央付近に集団で配置される場合と、通路に単独で配置される場合があります。

### 6.4 トラップの配置
- `trapSeed` に基づき、通路の入り口や、アイテムの周囲に優先的に配置されます。
- 初期状態ではすべて「隠蔽状態」となります。

## 7. 今後の拡張
- **フロアテーマの深化**: すでに定義されている「溶岩」「水路」や環境効果に加え、特定のテーマに基づいたタイルセットや専用オブジェクト（氷の床、動く床等）を自動配置する仕組み。
- **隠し部屋**: 通常の通路では繋がっておらず、壁を破壊したり隠しスイッチを押すことで入れる部屋。

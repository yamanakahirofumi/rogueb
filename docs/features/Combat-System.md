# 戦闘システム (Combat System)

## 1. 概要
本ドキュメントは、RogueBにおける戦闘システムの基本仕様を定義します。プレイヤーがダンジョン内でモンスターや他のプレイヤー（PK）と交戦する際のメカニズム、ダメージ計算、および状態異常について記述します。

## 2. 戦闘アクション
プレイヤーおよびエネミーは、自身のターンにおいて以下の戦闘アクションを選択できます。

### 2.1 通常攻撃 (Normal Attack)
- 隣接する 8 方向のいずれかに対して物理攻撃を行います。
- 装備している武器の性能に基づきダメージを算出します。

### 2.2 遠距離攻撃 (Ranged Attack)
- 矢や投擲アイテム、杖の魔法弾などを使用して、離れた地点の対象を攻撃します。
- 飛翔体は壁や他のエンティティに衝突するまで直進します。

### 2.3 スキル・魔法 (Skill & Magic)
- スタミナや MP を消費して、特殊な効果を発動します。
- 範囲攻撃、自己バフ、デバフ付与などが含まれます。
- 詳細は [スキル・魔法システム](./Skill-And-Magic-System.md) を参照してください。

## 3. ダメージ計算
ダメージの算出には以下の基本式を用います。

### 3.1 物理ダメージ
`ダメージ = (攻撃力 * 威力倍率 * 属性補正 * 乱数係数) - (防御力 / 2)`
- **威力倍率**: 通常攻撃は 1.0。スキルを使用する場合はスキルごとに設定された値（例：1.5）が適用されます。
- **攻撃力・防御力**: 装備品（武器・防具・指輪）による修正値を含みます。
- **属性補正**: 装備している武器に属性がある場合、後述の「属性相性表」に基づき 0.5 〜 2.0 倍の範囲で変動します。武器が「None（無属性）」の場合は常に 1.0 となります。
- **乱数係数**: 0.9 〜 1.1 の範囲で変動します。
- 算出結果が 1 未満の場合は、最低ダメージとして 1 が適用されます。

### 3.2 クリティカルヒット
- 一定の確率で発生し、最終ダメージを 1.5 倍にします。
- **発生確率(%)**: `攻撃側の器用さ / 2`
  - 最低 5%、最大 25% の範囲で制限されます。

### 3.3 魔法ダメージ
`ダメージ = (魔法攻撃力 * 威力倍率 * 属性補正 * 乱数係数) - (魔法防御力 / 2)`
- **威力倍率**: 魔法の種類ごとに設定された値（例：1.2）が適用されます。
- **属性補正**: 攻撃側の属性と防御側の耐性によって、以下の通り 0.5 〜 2.0 倍の範囲で変動します。
- **乱数係数**: 0.9 〜 1.1 の範囲で変動します。
- 物理ダメージと同様、最低ダメージとして 1 が適用されます。

#### 属性相性表
属性は「火 (Fire)」「水 (Water)」「風 (Wind)」「土 (Earth)」の4すくみの関係にあります。モンスターの属性は `MonsterDomain.attribute` に定義されます。

| 攻撃 \ 防御 | Fire | Water | Wind | Earth |
| :--- | :---: | :---: | :---: | :---: |
| **Fire** | 1.0 | 0.5 | 2.0 | 1.0 |
| **Water** | 2.0 | 1.0 | 1.0 | 0.5 |
| **Wind** | 0.5 | 1.0 | 1.0 | 2.0 |
| **Earth** | 1.0 | 2.0 | 0.5 | 1.0 |

- **有利 (2.0x)**: Fire > Wind, Wind > Earth, Earth > Water, Water > Fire
- **不利 (0.5x)**: Fire < Water, Water < Earth, Earth < Wind, Wind < Fire
- **それ以外 (1.0x)**: 同属性または対角属性（None 属性を含む）

### 3.4 命中判定 (Hit Rate)
物理攻撃および遠距離攻撃の命中率は以下の式で算出されます。
`命中率(%) = 95 + (攻撃側の器用さ - 防御側の器用さ) * 0.5`
- **最低命中率**: 50%
- **最高命中率**: 100%
- 魔法攻撃（魔法弾など）は、回避不能（命中率 100%）として扱われます。

### 3.5 経験値の獲得
モンスターを撃破した際、以下の式に基づき経験値を獲得します。
`獲得経験値 = 基本経験値 * (モンスターレベル / プレイヤーレベル) * 補正係数`
- **基本経験値**: `MonsterDomain` の `expValue`（基本経験値）に基づきます。
- **補正係数**: プレイヤーとモンスターのレベル差（PL - ML）に基づき、以下の通り変動します。

| レベル差 (PL - ML) | 補正係数 | 備考 |
| :--- | :---: | :--- |
| +10 以上 | 0.1 | 格下相手（大幅な減衰） |
| +5 〜 +9 | 0.5 | 格下相手 |
| -4 〜 +4 | 1.0 | 同格 |
| -9 〜 -5 | 1.2 | 格上相手（ボーナス） |
| -10 以下 | 1.5 | 強敵（大幅なボーナス） |

### 3.6 プレイヤーのレベルアップ
プレイヤーが一定の累積経験値を獲得すると、レベルが上昇します。

`累計必要経験値(L) = 10 * (L - 1)^2`
- L は目標とするレベルを表します（例: レベル2に到達するには 10、レベル3には 40 の累計経験値が必要）。
- レベルアップ時、最大 HP/MP が上昇し、HP/MP が全回復します。

#### ステータス成長の計算式
プレイヤーのレベルに応じた各ステータスは、モンスターと同様の式で算出されます。

`現在のステータス = 基本ステータス * (1 + (レベル - 1) * 0.1)`

- **基本ステータス**: `PlayerDomain.status` に保持されている値（Lv 1 時点の値）。
- **対象ステータス**: `atk`, `def`, `magicAtk`, `magicDef`, `dex`, `mnd`, `maxHp`, `maxMp`
- **スタミナ**: スタミナの最大値 (`maxStamina`) はレベルに関わらず固定（例: 100）ですが、装備品や特定の成長要素によって拡張される場合があります。

### 3.7 状態異常の付与判定
状態異常の付与成功率は、以下の式で算出されます。
`付与成功率(%) = 基本付与率 * (攻撃側の器用さ / 防御側の精神力)`
- **基本付与率**: スキルやアイテムごとに設定された値。
- **器用さ・精神力**: プレイヤーおよびモンスターのステータス。

## 4. ターン制と行動間隔
本作は基本的にはターン制ですが、行動後の待機時間によって次の行動までの間隔が決定されます。

### 4.1 行動間隔 (Action Interval)
`WaitTime (ms) = 基本行動間隔 (actionInterval) * 行動負荷係数`
- **基本行動間隔**:
    - プレイヤー: `currentStatus.actionInterval` に基づきます。
    - モンスター: `MonsterDomain.baseActionInterval` に基づきます。
- **行動負荷係数**: アクションごとに以下の係数が適用されます。

| アクション | 係数 | 備考 |
| :--- | :---: | :--- |
| 通常移動 / 足踏み待機 | 1.0 | 基本 |
| 通常攻撃 / 遠距離攻撃 | 1.2 | |
| アイテム使用（薬、食料等） | 1.0 | |
| 巻物を読む | 1.5 | |
| 杖を振る | 1.2 | |
| スキル（軽） | 1.5 | |
| スキル（重） | 2.0 | |

### 4.2 MP の自然回復
- プレイヤーおよびモンスターは、一定歩数（例：10歩）ごとに 1 MP 回復します。

### 4.3 スタミナ（満腹度）の消費と影響
プレイヤーは行動するごとにスタミナを消費します。
- **消費速度**: 10 歩（または 10 回の行動）ごとに 1 スタミナを消費します。
- **スタミナ上限**: `status.maxStamina` によって定義されます（初期値: 100）。
- **スタミナ切れ（0）のペナルティ**:
    - 自然回復（HP）が停止します。
    - 1 歩ごとに HP が 1 減少します。
- **回復**: 「パン」などの食料アイテムを使用することで回復します。

### 4.4 内部歩数カウンタ (subStep) と更新タイミング
スタミナ消費や継続ダメージの計算を正確に行うため、プレイヤー（`currentStatus`）およびモンスター（`MonsterInstanceDomain`）に `subStep`（内部歩数カウンタ）を導入します。

- **更新タイミング**:
    - プレイヤー：移動（通常移動、階段移動）、足踏み待機（Wait）、攻撃、スキル使用、およびアイテム使用を含む、**自身のターンを消費するすべてのアクション**において、`subStep` が +1 されます。
    - モンスター：自身の行動ターン（移動、攻撃、待機）ごとに `subStep` が +1 されます。
- **スタミナ消費（プレイヤーのみ）**: `subStep % 10 == 0` のタイミングでスタミナを 1 消費します。
- **自然回復 (HP)**:
    - プレイヤー：`subStep % 10 == 0` かつスタミナが 1 以上の場合、最大 HP の一定割合（例: 1%）を回復します。
    - モンスター：`subStep % 10 == 0` のタイミングで、最大 HP の一定割合（例: 1%）を回復します（スタミナの概念はありません）。
- **状態異常の更新**: 各アクション（移動、攻撃等）の終了時に、`StatusEffectDomain.remainingTurns` をデクリメントし、0 になった状態異常を解除します。

## 5. 状態異常 (Status Effects)
戦闘を有利・不利に進めるための状態異常を定義します。持続時間は `subStep`（歩数）または `ターン` 数（自身の行動回数）で管理されます。

| 名称 | 詳細効果 | 持続時間 | 解除条件 |
| :--- | :--- | :--- | :--- |
| **毒** | `subStep % 10 == 0` のタイミングで 2 ダメージ | 50 subStep | 自然治癒（時間経過）または薬草 |
| **混乱** | 移動・攻撃の方向が 8 方向ランダム | 20 ターン | 自然治癒 |
| **麻痺** | 25% の確率で行動がキャンセルされる | 15 ターン | 自然治癒 |
| **睡眠** | 行動不能（回避不可） | 10 ターン | 自然治癒または被ダメージ |
| **封印** | 杖・巻物・スキルの使用不可 | 30 ターン | 自然治癒または専用アイテム |
| **怒り** | 攻撃力が 1.5 倍になるが、防御力が半分になる | 20 ターン | 自然治癒 |
| **捕縛** | 移動不能（移動以外の行動は可能） | 10 ターン | 自然治癒 |
| **不可視** | モンスターから視認されなくなる（一部の敵を除く） | 30 ターン | 自然治癒 |

## 6. 死亡とペナルティ
HP が 0 になった場合、キャラクターは死亡します。
デスペナルティの詳細は、管理者が設定したルールに基づきます。
詳細は [機能仕様書](./Functional-Specification.md) の「4. デスペナルティの設定」を参照してください。

## 7. 実装案

### 7.1 モジュール間連携
戦闘発生時の基本的なデータフローは以下の通りです。

1. **PlayerOperations**: クライアントからの攻撃コマンドを受信。
2. **PlayerOperations -> Dungeon**: 攻撃範囲内にターゲットが存在するか確認。
3. **PlayerOperations -> Objects**: 装備品（武器・防具）の性能データを取得。
4. **PlayerOperations**: ダメージ計算を実行。
5. **PlayerOperations -> BookOfAdventure**: プレイヤー（またはターゲット）の HP/MP 更新を要求。
6. **PlayerOperations -> Display**: 戦闘ログおよび演出データを送信。

### 7.2 拡張性
- **属性（火、水、風、土など）**: 武器やモンスターに属性を付与し、相性によるダメージ増減を導入。
- **部位破壊**: 特定の強力なモンスターに対し、特定の部位を攻撃して無力化する要素の検討。

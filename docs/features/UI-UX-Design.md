# UI-UX 設計 (UI-UX Design)

## 1. 概要
本ドキュメントは、RogueBにおけるユーザーインターフェース（UI）およびユーザーエクスペリエンス（UX）の設計方針を定義します。本ゲームはマルチプレイヤー・ローグライクであり、探索と建築、そして経済活動をシームレスに行えるUIを目指します。

## 2. 画面構成
主要なゲーム画面は、以下の 4 つのエリアで構成されます。

### 2.1 メインビュー（ダンジョン表示エリア）
- **役割**: ダンジョンのタイル、プレイヤー、モンスター、アイテムを表示します。
- **表示形式**: グリッドベースのタイル表示。
- **視界制限**: プレイヤーの周辺（視界範囲内）のみが明るく表示され、それ以外は暗転（未探索）または半透明（既探索だが現在は見えない）となります。

### 2.2 ステータスバー（上部）
- **表示項目**:
  - プレイヤー名 / レベル
  - HP（現在値 / 最大値）のバー表示
  - スタミナ（空腹度等）
  - 所持ゴールド
  - 現在の階層（例: B5F）

### 2.3 メッセージログ（下部）
- **役割**: ゲーム内で発生したイベント（攻撃、アイテム拾得、階段昇降など）をテキストで通知します。
- **UX**: 重要なイベント（ダメージを受けた、レベルアップ等）は色を変えて強調します。

### 2.4 サイドパネル / メニュー
- **役割**: 状況に応じて以下の情報を表示します。
  - **ミニマップ**: フロア全体の探索状況。
  - **インベントリ（Bag）**: 所持アイテムの一覧と操作（使う、捨てる、装備）。
  - **モンスター情報**: 捕獲・所持しているモンスターの状態。

## 3. 操作フィードバック
- **移動**: キー入力に対し、即座に座標が更新されます（リアクティブなレスポンス）。
- **戦闘**: 攻撃ヒット時に画面の揺れ（スクリーンシェイク）や、ダメージ数値のポップアップを表示します。
- **アイテム**: 拾得時にメッセージログへの表示に加え、効果音（SE）によるフィードバックを行います。

## 4. 特殊画面の UI 設計

### 4.1 ダンジョンエディタ（管理者モード）
- ツールボックス形式でタイルやモンスターを選択し、マップ上に配置する UI。
- グリッドガイドの表示。

### 4.2 ショップ UI
- 店主の在庫リストとプレイヤーの所持アイテムを並列表示。
- 基本価格と現在価格（管理者が設定）を比較できるインターフェース。

## 5. デザイン規約
- **フォント**: 視認性の高い等幅フォント（Monospaced Font）を推奨。
- **カラーパレット**:
  - 壁: 暗いグレー
  - 床: 明るいグレー / ベージュ
  - プレイヤー: 黄色 / 白色
  - モンスター: 赤色 / 紫色
  - ゴールド: 金色
